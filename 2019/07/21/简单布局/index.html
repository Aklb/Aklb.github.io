简单布局

发表于 2019-07-21 更新于 2019-08-22 分类于 Android 阅读次数：

Android的基本视图和布局，首先是基本视图View类的常用属性和方法，接着是如何使用线性布局LinearLayout，最后介绍滚动视图ScrollView的用法。

视图View的基本属性

View是Android的基本视图，所有控件和布局都是由View类直接或间接派生出来的——View类的基本属性和方法是各个控件和布局通用的。

一个丰富的界面总是要由很多个控件组成的，那我们如何才能让各个控件都有条不紊地摆放
在界面上，而不是乱糟糟的呢？这就需要借助布局来实现了。布局是一种可用于放置很多控件的
容器，它可以按照一定的规律调整内部控件的位置，从而编写出精美的界面。当然，布局的内部
除了放置控件外，也可以放置布局，通过多层布局的嵌套，我们就能够完成一些比较复杂的界面
实现，图 3.15很好地展示了它们之间的关系。（《第一行代码(第二版)》P94）

视图在XML布局文件中常用的属性定义说明：

  • id：指定该视图的编号

  • layout_width：指定该视图的高度

    1. 可以是具体的dp数值
    2. 可以是match_parent：表示与上级视图一样宽
    3. 也可以是warp_content：表示与内部内容一样宽（内部内容若超过上级视图的宽度，则该视图保持与上级视图一样宽，超出宽度的内容的进行滚动才能显示出来）

  • layout_height：指定该视图的高度。取值同layout_width

  • layout_margin：指定该视图与周围视图之间的空白距离（包括上下左右）

    另有layout_marginTop、layout_marginBottom、layout_marginLeft、layout_marginRight分别表示单独指定视图与上边、下边、左边、右边视图的距离。

  • minWidth：指定该视图的最小宽度

  • minHeight：指定该视图的最小高度

  • background：指定该视图的背景——可以是颜色，也可以是图片

  • layout_gravity：指定该视图与上级视图的对齐方式。对齐方式的取值说明见下表，若同时使用多种对齐方式，则需要使用竖线“|”把多种对齐方式拼接起来

    下面是视图在代码中常用的设置方法说明：

  • setLayoutParams：设置该视图的布局参数。参数对象的构造函数可以设置视图的宽度和高度

    1. LayoutParams.MATCH_PARENT表示与上级视图一样宽

    2. LayoutParams.WRAP_CONTENT，表示与内部内容一样宽

      参数对象的setMargins方法可以设置该视图与周围视图之间的空白距离。

  • setMinimumWidth：设置该视图的最小宽度

  • setMinimumHeight：设置该视图的最小高度

  • setBackgroundColor：设置该视图的背景颜色

  • setBackgroundDrawable：设置该视图的背景图片

  • setBackgroundResource：设置该视图的背脊资源ID

  • setPadding：设置该视图边缘与内部内容之间的空白距离

  • setVisibility：设置该视图的可视类型。取值见可视类型的取值说明表（上个表格）

注意：

  1. margin：指的是当前视图与周围视图的距离

  2. padding：指的是当前视图与内部内容的距离

    视图组ViewGroup是一类特殊视图，所有的布局文件都是从它派生出来的。

    Android中的视图分为两类：一类是布局，另一类是控件。

    布局与控件的区别：

    1. 布局本质上是个容器，里面还可以放其他视图（包括子布局和子控件）

    2. 控件是一个单一的实体，已经是最后一级，下面不能再挂其他视图。

      可以看到，我们所用的所有控件都是直接或间接继承自 View 的，所用的所有布局都是直接
      或间接继承自 ViewGroup的。View是 Android中最基本的一种 UI组件，它可以在屏幕上绘制一
      块矩形区域，并能响应这块区域的各种事件，因此，我们使用的各种控件其实就是在 View 的基
      础之上又添加了各自特有的功能。而 ViewGroup则是一种特殊的 View，它可以包含很多子 View
      和子 ViewGroup，是一个用于放置控件和布局的容器。（《第一行代码(第二版)》P109）

ViewGroup有三个方法，这三个方法也是所有布局类视图共同拥有的。

  • addView：往布局中添加一个视图

  • removeView：从布局中删除指定视图

  • removeAllView：删除该布局下的所有视图

    线性布局LinearLayout

    LinearLayout是最最常用的布局，名字叫线性布局。LinearLayout下面的子视图，就像用一根线串起来，所以LinearLayout内部视图的排列都是有序的，要么从上往下依次垂直排列，要么从左到右依次水平排列。LinearLayout除了继承View/ViewGroup类的所有属性和方法外，还有其特有的属性，说明如下：

    • orientation：指定线性布局的方向。

      1. horizontal：表示水平布局

      2. vertical：表示垂直布局

        注意：如果不指定该属性，默认的是vertical

    • gravity：指定布局内部视图与本线性布局的对齐方式。取值说明同layout_gravity

    • layout_weight：指定当前视图与的宽或高占上级线性布局的权重。
