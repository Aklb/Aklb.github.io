像素

Android支持的像素单位有：px（像素）、in（英寸）、mm（毫米）、pt（磅，1/27英寸）、dp（与设备无关的显示单位）、dip（就是dp）、sp（用于设置字体的大小）。

常用的有：px、dp、sp

单位
说明
px
是手机上可显示的最小单位，与物理设备有关
dp
与物理设备无关，只与屏幕尺寸有关
sp
原理跟dp差不多，专门用于设置字体大小

​注意：​dp与系统设置的大小没有关系，而sp会随着系统设置字体大小变大或变小。

​ dp和px之间的联系取决于具体设备上的像素密度，像素密度就是DisplayMetrics里的density参数。

​ 当density=1.0时，一个dp值对应一个px值；
​ 当density=1.5时，两个dp值对应三个px值；
​ 当density=2.0时，一个dp值对应两个px值。

颜色

Android中，颜色值由透明度alpha和RGB（红、绿、蓝）三原色定义——有八位十六进制数与六位十六进制数两种编码。

eg：八位编码FFEEDDCC

FF表示透明度

EE表示红色的浓度

DD表示绿色浓度

CC表示蓝色浓度

透明度（浓度）：FF——表示完全不透明；00——表示完全透明

RGB——三色的数值越大，颜色越浓也就越亮，数值越小颜色就越暗。助记：亮到极致就是白，暗到极致就是黑

注意：六位十六进制编码有两种情况，在XML文件中默认不透明（透明度为FF），在代码中默认透明（透明度为00）。

2.使用十六进制的颜色编码

在布局文件中，设置颜色需要在颜色值前面加“#”。eg：android:textColor=”#000000”

在代码中，设置颜色，可以直接写八位十六进制数值。eg：setTextColor(0xff00ff00)；，也可以通过Color.rgb(int red,int green,int blue)和Color.argb(int alpha,int red,int green,int blue)这两种方式指定颜色。在代码中一般不要用六位编码——因为六位编码在代码中默认透明，所以代码用六位编码和不用没有什么区别。

3.使用colors.xml中定义颜色

res/values目录下有个colors.xml文件，是颜色常量的定义文件。

如果要在布局文件中使用XML颜色常量，可引用“@color/常量名”；

如果要在代码中使用XML颜色常量，可通过这行代码获取：getResources().getColor(R.color.常量名)。

屏幕分辨率

在App编码中经常需要获取当前设备屏幕分辨率（当前屏幕的高和宽），然后动态调整界面布局。在代码中获取分辨率就是想办法得到DisplayMetrics对象，然后从该对象中获得宽度、高度、像素密度等信息。

下面是DisplayMetrics类的常用属性说明：

  • widthPixels：以px为单位计量的宽度值

  • heightPixels：以px为单位计量的高度值

  • density：像素密度，即一个dp单位包含多少个px单位
